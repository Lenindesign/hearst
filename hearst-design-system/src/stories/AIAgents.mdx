## Skill File Structure

Each agent is defined as a Cursor skill in the project:

```
.cursor/skills/
  hearst-token-architect/SKILL.md      (67 lines)
  hearst-figma-sync/SKILL.md           (77 lines)
  hearst-pencil-design/SKILL.md        (90 lines)
  hearst-frontend-component/SKILL.md   (81 lines)
  hearst-storybook-docs/SKILL.md       (100 lines)
  hearst-devops-deploy/SKILL.md        (86 lines)
  hearst-qa-review/SKILL.md            (113 lines)
```

Skills are **auto-discovered** by Cursor based on their `description` field. When you ask a question or request a task, the matching agent's knowledge is automatically loaded into context.

### What Each Skill Contains

- **Frontmatter** -- `name` and `description` (triggers auto-discovery)
- **File paths** -- Exact files the agent owns and operates on
- **Scripts** -- npm commands and their purpose
- **Conventions** -- Naming rules, patterns, known issues
- **Rules** -- Guardrails to prevent mistakes (e.g. "never edit generated files")
