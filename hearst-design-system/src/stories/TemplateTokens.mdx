## Variable Reference

Every CSS custom property available to templates, grouped by category. These are defined in `globals.css` and consumed via Tailwind's arbitrary value syntax (e.g. `text-[length:var(--text-token-md)]`).

## Token Categories at a Glance

Both templates draw from the same five token families. The difference is *how* they use them — the home page prioritizes **density and scanning**, while the article page prioritizes **readability and focus**.

### Why `.headline` matters

The `.headline` class applies `font-family: var(--font-headline)` with the brand's headline weight. This is the single most important brand differentiator — it's what makes Cosmopolitan look different from Esquire. Both templates use it for their most prominent text, but the article page applies it more broadly (subheadings, pull quotes) because the reading experience needs more typographic variety.

## Rules for New Templates

1. **Never hardcode pixel values** — use `--text-token-*` for typography and `--spacing-token-*` for spacing
2. **Always use `--width-content-max`** for the main content constraint — this ensures consistent page width
3. **Use `--width-sidebar`** for any right rail — ads and sidebar content share this standard width
4. **Apply `.headline`** to the most prominent text in each section — this is the brand's typographic signature
5. **Use `font-brand-secondary`** for functional labels (eyebrows, breadcrumbs, section headers) — it provides contrast with the headline font
6. **Color tokens come from the theme** — never reference brand colors directly; use `bg-primary`, `text-foreground`, etc.
7. **Reading templates need `--text-token-md` (18px) minimum** for body text — scanning templates can use `--text-token-sm` (16px) or smaller
8. **Spacing should increase with reading depth** — home page (scanning) uses tighter spacing; article page (reading) uses more generous spacing
