# Hearst Design System

## Architecture

The system follows a strict one-way data flow. Git is the single source of truth — everything else is derived.

```
                    ┌─────────────────────────────┐
                    │     Git  tokens/             │
                    │     (single source of truth) │
                    └──────────┬──────────────────┘
                               │
              ┌────────────────┼────────────────┐
              │                │                │
              ▼                ▼                ▼
        build-tokens      push-figma      push-pencil
              │                │                │
              ▼                ▼                ▼
        brands.ts          Figma           Pencil
        tokens.css         Variables       Variables
              │
     ┌────────┴────────┐
     │                 │
     ▼                 ▼
  Next.js App      Storybook
  (main site)      (component docs)
     │                 │
     ▼                 ▼
  Netlify           Netlify
  (auto-deploy)     (separate site)
```

## Workflow at a Glance

<div style={{ display: 'grid', gridTemplateColumns: '1fr 1fr', gap: '16px', margin: '24px 0' }}>
  <div style={{ border: '1px solid #e2e8f0', borderRadius: '12px', padding: '20px', background: '#fafbfc' }}>
    <h4 style={{ margin: '0 0 8px', fontSize: '14px', color: '#0f172a' }}>Designer changes a token value</h4>
    <ol style={{ margin: 0, paddingLeft: '20px', fontSize: '13px', color: '#475569', lineHeight: '2' }}>
      <li>Open Cursor, describe the change in natural language</li>
      <li>AI creates a branch, edits the JSON, runs validation</li>
      <li>AI opens a PR — developer reviews and merges</li>
      <li>Developer runs <code>push-figma</code> + <code>push-pencil</code></li>
      <li>Designer verifies in Figma Variables panel</li>
    </ol>
  </div>
  <div style={{ border: '1px solid #e2e8f0', borderRadius: '12px', padding: '20px', background: '#fafbfc' }}>
    <h4 style={{ margin: '0 0 8px', fontSize: '14px', color: '#0f172a' }}>Developer adds a new component</h4>
    <ol style={{ margin: 0, paddingLeft: '20px', fontSize: '13px', color: '#475569', lineHeight: '2' }}>
      <li>Create tokens in <code>tokens/</code> JSON if needed</li>
      <li>Run <code>build-tokens</code> to generate CSS variables</li>
      <li>Build the React component using <code>var(--token)</code></li>
      <li>Add a Storybook story with controls and docs</li>
      <li>PR, merge, deploy — Netlify auto-deploys both sites</li>
    </ol>
  </div>
</div>

## The Golden Rules

<div style={{ display: 'grid', gridTemplateColumns: '1fr 1fr', gap: '16px', marginTop: '16px' }}>
  <div style={{ border: '2px solid #22c55e', borderRadius: '12px', padding: '20px', background: '#f0fdf4' }}>
    <h4 style={{ margin: '0 0 12px 0', color: '#16a34a', fontSize: '14px' }}>DO</h4>
    <ul style={{ margin: 0, paddingLeft: '20px', fontSize: '13px', lineHeight: '1.9', color: '#334155' }}>
      <li>Edit tokens in <strong>Git</strong> — it's the single source of truth</li>
      <li>Explore designs in <strong>Figma</strong></li>
      <li>Build approved specs in <strong>Pencil</strong></li>
      <li>Review components in <strong>Storybook</strong></li>
      <li>Run <code>build-tokens</code> after every token edit</li>
      <li>Run <code>push-figma</code> + <code>push-pencil</code> after merging</li>
    </ul>
  </div>
  <div style={{ border: '2px solid #ef4444', borderRadius: '12px', padding: '20px', background: '#fef2f2' }}>
    <h4 style={{ margin: '0 0 12px 0', color: '#dc2626', fontSize: '14px' }}>DON'T</h4>
    <ul style={{ margin: 0, paddingLeft: '20px', fontSize: '13px', lineHeight: '1.9', color: '#334155' }}>
      <li>Edit <code>brands.ts</code> or <code>tokens.css</code> directly</li>
      <li>Edit Figma Variables directly (they get overwritten)</li>
      <li>Hardcode hex values in Pencil (use <code>$variable</code>)</li>
      <li>Use Figma inspect values as source of truth</li>
      <li>Commit directly to <code>main</code> — always use a branch</li>
    </ul>
  </div>
</div>
