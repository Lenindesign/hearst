## The Canonical Name: One Name to Rule Them All

Designers and developers only need to know **one name** for each token -- the **canonical name**. This is the plain, hyphenated key from the Git JSON files, without any prefix or syntax.

### What a typography preset value looks like

Each typography token resolves to a JSON string with these properties:

```json
{
  "fontSize": "1.25rem",
  "fontFamily": "Chronicle Display",
  "lineHeight": "120%",
  "letterSpacing": "0px",
  "fontWeight": "600"
}
```

Some presets also include `"textCase": "uppercase"` for labels and badges.

### How typography tokens compose

```
typography-component-card-core-content-title-default-lg
    |
    references --> typography-size-lg-semibold
                       |
                       references --> font-size-lg       (1.25rem)
                                      font-line-height-lg (140%)
                                      font-letter-spacing-lg (0px)
                                      font-weight-semibold (600)
                                      font-family-default  (brand-specific)
```

<div style={{ display: 'grid', gridTemplateColumns: 'repeat(auto-fit, minmax(340px, 1fr))', gap: '24px', margin: '24px 0' }}>
  <div style={{ border: '1px solid #bbf7d0', borderRadius: '12px', padding: '20px', background: '#f0fdf4' }}>
    <h4 style={{ margin: '0 0 12px', color: '#16a34a' }}>DO</h4>
    <DoItem>Use <strong>kebab-case</strong> for all token names</DoItem>
    <DoItem>Follow the segment order: layer - category - element - variant - style - state - size</DoItem>
    <DoItem>Use existing size names: <code>xs, sm, md, lg, xl, 2xl, 3xl, 4xl, 6xl</code></DoItem>
    <DoItem>Use existing state names: <code>default, hover, active, error, knockout, selected</code></DoItem>
    <DoItem>Use existing variant names: <code>primary, secondary, neutral, brand, danger, success, warning</code></DoItem>
    <DoItem>Reference <code>$brand-1</code> in semantic aliases for brand-aware tokens</DoItem>
    <DoItem>Run <code>build-tokens</code> after every edit</DoItem>
    <DoItem>Check at least 3 brands after adding tokens</DoItem>
  </div>
  <div style={{ border: '1px solid #fecaca', borderRadius: '12px', padding: '20px', background: '#fef2f2' }}>
    <h4 style={{ margin: '0 0 12px', color: '#dc2626' }}>DON'T</h4>
    <DontItem>Invent new size names (use the established scale)</DontItem>
    <DontItem>Use camelCase or PascalCase (<code>buttonBackground</code>)</DontItem>
    <DontItem>Hardcode brand-specific hex values in <code>core/global.json</code></DontItem>
    <DontItem>Skip the <code>component-</code> prefix for component-scoped tokens</DontItem>
    <DontItem>Put shared values in brand files (use <code>global.json</code> instead)</DontItem>
    <DontItem>Edit <code>brands.ts</code> or <code>tokens.css</code> directly (they are generated)</DontItem>
    <DontItem>Create duplicate tokens with slightly different names</DontItem>
    <DontItem>Use abbreviations that are not already established</DontItem>
