# Editorial Modes

<div style={{ fontSize: '16px', color: '#475569', lineHeight: '1.7', marginBottom: '32px', maxWidth: '680px' }}>
From our head of design: homepage layouts aren't fixed identities — they're <strong>editorial states</strong>.
A brand can shift between modes based on purpose and timing, like an editorial operating system
with different "gears" for different jobs.
</div>

<Callout emoji="💡" title="Key Insight: Modes, not Archetypes">
The four layouts (Curator, Mosaic, Stream, Editorial) are structural templates.
Editorial Modes are the <strong>behavioral layer</strong> on top — the "jobs to be done"
for each variation of a UX pattern. A brand could identify with 2 of these at any given time,
or change modes from M–F versus Sat/Sun. They're less identities and more states.
</Callout>

## The Three Modes

<ModeCard name="Live Mode" tagline="What's happening now" color="#dc2626" values={["Recency", "Density", "Scanning"]} layout="The Curator">

Optimized for the **breaking news mindset**. High story count (7 top stories), prominent trending bar,
frequent ad slots, no inline newsletter (sticky CTA instead). The page feels *busy on purpose* —
every pixel is working to surface what's new.

**When to activate:** Weekday mornings, breaking news cycles, awards seasons, election nights, game days.

**Suppresses:** Inline newsletter (uses sticky CTA instead), shopping carousel, video spotlight.
These slow down scanning and add cognitive load when the user's job is "catch me up."

</ModeCard>

<ModeCard name="Feature Mode" tagline="Stories worth your time" color="#7c3aed" values={["Storytelling", "Visual", "Immersive"]} layout="The Editorial">

Optimized for the **lean-back reader**. Fewer stories (3 top stories), no leaderboard ad, no trending bar,
prominent newsletter, video spotlight enabled. The page breathes — hero imagery dominates and
longform content gets the space it deserves.

**When to activate:** Weekend editions, editorial showcases, magazine launches, fashion weeks,
long-form feature drops.

**Suppresses:** Leaderboard ads, trending bar, mid-page ads, sticky newsletter.
Ads and urgency signals undercut the immersive reading experience this mode is designed to create.

</ModeCard>

<ModeCard name="Service Mode" tagline="Find what you need" color="#0891b2" values={["Task", "Search", "Structured"]} layout="The Stream">

Optimized for the **goal-oriented visitor**. Shopping carousel enabled, high thematic row count (6),
more editor's picks (8), no big story feed (replaced by structured categories).
The page is organized like a tool, not a magazine.

**When to activate:** Always-on for utility brands, product roundup days, gift guide seasons,
recipe-heavy periods, "best of" compilations.

**Suppresses:** Big story feed, sticky newsletter, video spotlight.
Narrative content competes with the structured navigation these users need.

</ModeCard>

## Modes are States, Not Identities

<div style={{ fontSize: '14px', color: '#475569', lineHeight: '1.7', marginBottom: '24px' }}>
The critical mental shift: brands don't <em>have</em> a mode. They <em>enter</em> a mode.
A single brand can operate in multiple modes across a week, or even within a single day.
</div>

## Mode × Layout Matrix

Every mode has preferred layouts, but any mode can use any layout. The mode configures *what* appears and *how dense* it is; the layout determines *how* it's arranged.

## How to Test Modes in Storybook

Modes are available as stories under **Templates / Homepage Layouts**:

1. **Mode: Live** — Curator layout with Live presets applied
2. **Mode: Feature** — Editorial layout with Feature presets applied
3. **Mode: Service** — Stream layout with Service presets applied
4. **Mode Switcher** — Select any mode + optionally override the layout

Combine with the **brand switcher** in the Storybook toolbar to test real scenarios:
- "Esquire in Live Mode on a Tuesday"
- "Good Housekeeping in Service Mode"
- "Elle in Feature Mode during Fashion Week"

All section toggles remain available as manual overrides in the controls panel.

---

## Implementation

```tsx
// Render a homepage in a specific editorial mode
<HomepageByMode mode="live" />

// Override the default layout while keeping mode presets
<HomepageByMode mode="feature" layoutOverride="mosaic" />

// Override specific props on top of the mode preset
<HomepageByMode mode="service" topStoriesCount={8} showVideoSpotlight />

// Resolve a mode to its layout + props for custom logic
const { layout, props } = resolveMode("live");
```
