# The Toolbox

Think of the Hearst Design System as a **workshop**. Every craftsperson needs the right tool for the right job.
You wouldn't use a hammer to measure a board, and you wouldn't use a ruler to drive a nail.

Each tool in our system has a **specific purpose**, a **specific audience**, and a **specific moment** when it's the right one to reach for.

## Git (`tokens/`) — The Blueprint Vault

**Analogy:** The locked filing cabinet where the official blueprints are stored. Every measurement, every color, every specification lives here. If there's ever a disagreement about what the "right" value is, you open this cabinet.

The `tokens/` directory in the Git repository. JSON files containing every design token value — colors, spacing, fonts, borders, component-specific values — for all 29 brands.

**When to reach for it:**
- You need to **add, change, or remove** a design token (developers and designers)
- You need to **verify** the correct value of any token
- You need to **review the history** of when and why a token changed
- You need to **roll back** a token change that caused issues

**What makes it special:**
- Full version history — who changed what, when, and why
- Code review via pull requests — changes are reviewed before going live
- Rollback with `git revert` — instant undo
- Single source of truth — everything else is derived from here

```
tokens/
├── core/global.json           # 401 global tokens
├── semantic/aliases.json      # Semantic references
└── brands/
    ├── _meta.json             # Font overrides
    ├── cosmopolitan.json      # 595 tokens per brand
    ├── car-and-driver.json
    └── ... (29 brand files)
```

**When to reach for it:**
- You're **exploring** a new design concept
- You need to **iterate** on layouts, colors, or typography
- You're **prototyping** an interaction for user testing
- You need **stakeholder review** and approval
- You want to **collaborate** visually with other designers

**When NOT to reach for it:**
- To get the "official" hex value of a color (use Git tokens)
- To define new token values (request through the team)
- To hand off specs to developers (use Pencil)

**How it stays in sync:**

```
Git tokens/  ──── npm run push-figma ────►  Figma Variables
                                             (29 brand modes)
```

Designers see production tokens as Figma Variables, but they **cannot edit them back** — it's a one-way push.
