{/* ─── THE BIG IDEA ─── */}

<Section subtitle="The big idea" title="Git is the single source of truth">

Everything starts and ends in Git. Design tokens — colors, fonts, spacing, border radii — are stored as JSON files in a three-layer hierarchy. Every other tool in the system is a **consumer** of these files, not a source.

This means:
- Figma does not own the tokens. Git does.
- Pencil does not own the tokens. Git does.
- The deployed sites do not own the tokens. Git does.

When a designer or an AI agent changes a token value, they change it in Git. Then scripts push that change outward to every surface.

</Section>

When <Code>npm run build-tokens</Code> runs, it flattens these three layers into:
- <Code>src/lib/tokens.css</Code> — CSS custom properties for the Next.js app
- <Code>src/lib/brands.ts</Code> — TypeScript metadata for the ThemeProvider

These generated files are never edited by hand. They are rebuilt from the JSON source.

{/* ─── HOW AGENTS WORK ─── */}

<Section subtitle="Agentic infrastructure" title="Seven agents, one system">

Each agent is a <Code>SKILL.md</Code> file stored in <Code>.cursor/skills/</Code>. When a task matches an agent's domain, Cursor loads that skill file and the AI gets deep context: which files to touch, which scripts to run, which rules to follow, and which mistakes to avoid.

This is not generic AI assistance. Each agent knows the Hearst system specifically.

{/* ─── SELF-AWARENESS ─── */}

<Section subtitle="Instrumentation" title="The system knows itself">

Three scripts give the system self-awareness:

**Token Auditor** (<Code>npm run audit</Code>)
Scans every component file for hardcoded hex colors, non-semantic Tailwind classes, and raw color functions. Reports violations with line numbers, severity, and suggestions for which token to use instead. Currently tracking 115 violations across 17 files.

**Codebase Indexer** (<Code>npm run index</Code>)
Maps every component's exports, dependencies, token usage, atomic level, brand-awareness, and responsiveness. Generates a full dependency graph: which components use which, and which tokens go where. 65 components indexed.

**Component Metadata** (<Code>.metadata.ts</Code> files)
Each component can have a metadata file that tells AI agents: what the component does, when to use it, when not to, which tokens it consumes, what its variants and slots are, and any known limitations. This is the machine-readable spec that makes agents reliable.

Together, these three systems mean an agent can: read the index to find the right component, read the metadata to understand how to use it, and run the audit to verify it followed the rules.

</Section>

{/* ─── FOR DESIGNERS ─── */}

<Section subtitle="For designers" title="What this means for you">

You do not need to know React, TypeScript, or Git commands. Here is how you work:

1. **Open Cursor.** It is an editor, but you use it like a conversation.
2. **Describe what you want.** "Change Cosmopolitan's primary color to a deeper red." Or "Build a carousel that matches this Pencil spec."
3. **The agent does the rest.** It edits the right JSON file, rebuilds tokens, pushes to Figma and Pencil, runs the audit, and deploys.
4. **You verify.** Open the Storybook link or the dev server. Switch brands. Check that it looks right.
5. **If something is off, say so.** "The spacing is too tight on mobile" — the agent adjusts and redeploys.

The manifesto says: **if you didn't test it in the product, you didn't design it.** This system makes that possible without writing code.

</Section>

{/* ─── FOR DEVELOPERS ─── */}

<Section subtitle="For developers" title="What this means for you">

The codebase is a standard Next.js 16 + React 19 + Tailwind CSS 4 project. What makes it different:

- **Tokens are first-class data.** Not scattered across Figma files or Notion pages. They are typed JSON files with a build pipeline.
- **Components are instrumented.** Metadata files, the dependency index, and the token auditor give you a machine-readable map of the system.
- **Agents have guardrails.** SKILL.md files encode rules like "never hardcode colors" and "always test 3 brands." The agent follows them before you have to review.
- **The official HDS lives at** <a href="https://github.com/Media-Platforms/hearst-design-system" style={{ color: '#3b82f6' }}>Media-Platforms/hearst-design-system</a>. This repo is the testing and iteration environment where changes are proven before they move upstream.

</Section>
