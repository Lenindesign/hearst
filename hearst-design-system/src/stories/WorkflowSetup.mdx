# Workflow setup: Figma + Git + Pencil Together

Use this page when you are **onboarding** or explaining the system to someone new. It answers: *“Where do I work, what do I install, and how do the three tools stay in sync?”*

Designers and developers share one rule: **Git (`tokens/`) is the single source of truth.** Figma and Pencil are **consumers** of that truth — they receive updates via scripts; they do not send token values back into Git.

**Flow (one direction):**

```
  tokens/  ──►  npm run build-tokens   ──►  brands.ts, tokens.css  (this app: Next.js + test Storybook)
  tokens/  ──►  npm run push-figma      ──►  Figma Variables (29 brand modes)
  tokens/  ──►  npm run push-pencil     ──►  Pencil variables in hearst-brands.pen
```

After a merge to `main`, a developer runs `build-tokens` (for the app) and, when needed, `push-figma` and `push-pencil` so designers see the same names and values in Figma and Pencil. Shipping to **production** follows your team’s release process into the canonical repo ([Media-Platforms/hearst-design-system](https://github.com/Media-Platforms/hearst-design-system)) and any downstream consumers (e.g. FRE).

## Designer setup

**Goal:** You can explore in Figma, build specs in Pencil, and change token values **through Git** (with review), not by editing Figma variables directly.

1. **Access**
   - Figma: edit access to the Hearst tokens file (ask your team lead).
   - **Pencil:** installed and signed in; open `hearst-brands.pen` from this repo when you work on handoff.
   - **Cursor** (optional but recommended): clone the repo so you can edit `tokens/` on a branch with AI assistance, or pair with a developer.

2. **Learn the names**
   - Read [Token Naming Guide](?path=/docs/token-naming-guide--docs) for canonical paths.
   - [Token Workflow](?path=/docs/token-workflow--docs) describes how requests and PRs work.

3. **Daily workflow**
   - **Ideate** in Figma using variables from the synced collection; switch **modes** to preview brands.
   - **Lock approved specs** in Pencil using `$variable-name` — never hardcode hex for production tokens.
   - **Change a value:** either file a request for a developer, or edit JSON in Cursor on a branch and open a PR (see Token Workflow).

4. **Verify a sync**
   - In Figma: Variables panel → find the name the team agreed on; check the mode matches your brand.
   - In Pencil: variables should update after `push-pencil`; ask a developer if names/values look stale.

## Developer setup

**Goal:** You know where **official** HDS lives on GitHub, and you can run **this** repo locally to build tokens and push to Figma and Pencil.

1. **Official Hearst Design System (day-to-day integration)**

   - **Canonical repo:** [Media-Platforms/hearst-design-system](https://github.com/Media-Platforms/hearst-design-system)
   - **Storybook:** Use your team’s **official** HDS Storybook (published from that repo or its CI), not only this repository’s Storybook, when you need the supported component API.
   - **FRE (optional):** [apps/fre/scopes/hdm/hds](https://github.com/Media-Platforms/fre/tree/main/apps/fre/scopes/hdm/hds) in [Media-Platforms/fre](https://github.com/Media-Platforms/fre) if your squad ships HDS through that app.
   - This local app remains a **sandbox** for token experiments, docs, and tests before changes land in the org repo.

2. **This repository (testing & token pipeline)**

   ```bash
   git clone <this-repo-url>
   cd hearst/hearst-design-system
   npm install
   ```

3. **Local apps (this repo only — testing)**

   - Next.js: `npm run dev` → [http://localhost:3000](http://localhost:3000)
   - Storybook: `npm run storybook` → [http://localhost:6006](http://localhost:6006) — **official** component catalog and MDX docs (Hearst logo in the shell).

4. **Token pipeline**
   - After editing anything under `tokens/`: `npm run build-tokens`
   - Validate: `npm run tokens:validate`

5. **Push to Figma** (requires env vars)

   Set these in your shell or `.env` (never commit secrets):

   - `FIGMA_TOKEN` — personal access token with file write access  
   - `FIGMA_FILE_KEY` — from the Figma file URL (e.g. `.../design/FILE_KEY/...`)

   Then run:

   ```bash
   npm run push-figma
   ```

   Details: [Figma Integration](?path=/docs/figma-integration--docs)

6. **Push to Pencil**

   ```bash
   npm run push-pencil
   ```

   Pencil MCP in Cursor is used to read/edit `.pen` content; the script syncs variables from Git. Details: [Pencil Integration](?path=/docs/pencil-integration--docs)

7. **Do not edit generated files**
   - Do not hand-edit `tokens.css` or generated `brands.ts` — always change `tokens/` and run `build-tokens`.
